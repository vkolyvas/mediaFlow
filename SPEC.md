## What We're Building

A synchronous web app that:
1. Accepts a pasted transcript
2. Extracts semantic content (hooks, claims, stories, data)
3. Applies a user-selected template style (AIDA, PAS, Story, etc.)
4. Generates draft posts for LinkedIn, X, TikTok, and Reddit simultaneously
5. Presents all drafts in a review dashboard where user can edit and approve

Stack: **Next.js (frontend) + FastAPI (backend) + SQLite (local) + Claude API**

No Redis. No RQ. No async workers. Everything runs synchronously in-process.
Reddit OAuth is v2 — MVP generates draft text only, no posting.

### Step 2 — Database schema (SQLite)

```sql
CREATE TABLE IF NOT EXISTS projects (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS transcripts (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    raw_text TEXT NOT NULL,
    cleaned_text TEXT,
    word_count INTEGER,
    created_at TIMESTAMPTZ DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS semantic_chunks (
    id TEXT PRIMARY KEY,
    transcript_id TEXT REFERENCES transcripts(id),
    chunk_type TEXT,
    content TEXT NOT NULL,
    confidence_score FLOAT,
    created_at TIMESTAMPTZ DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS template_styles (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    structure TEXT NOT NULL  -- JSON string of TemplateSection[]
);

CREATE TABLE IF NOT EXISTS generation_runs (
    id TEXT PRIMARY KEY,
    project_id TEXT REFERENCES projects(id),
    transcript_id TEXT REFERENCES transcripts(id),
    template_id TEXT REFERENCES template_styles(id),
    status TEXT DEFAULT 'pending',
    created_at TIMESTAMPTZ DEFAULT (datetime('now')),
    completed_at TIMESTAMPTZ
);

CREATE TABLE IF NOT EXISTS generated_posts (
    id TEXT PRIMARY KEY,
    generation_run_id TEXT REFERENCES generation_runs(id),
    platform TEXT NOT NULL,
    content TEXT NOT NULL,
    quality_score FLOAT,
    reviewed INTEGER DEFAULT 0,
    approved INTEGER DEFAULT 0,
    edited_content TEXT,
    created_at TIMESTAMPTZ DEFAULT (datetime('now'))
);
```

Note: `pgvector VECTOR(1536)` column removed — embeddings deferred to v2. `TIMESTAMPTZ` uses SQLite's `TEXT` storage with ISO format.

### Step 4 — Provider abstraction layer

```python
# app/ai/base.py
from abc import ABC, abstractmethod

class LLMProvider(ABC):
    @abstractmethod
    async def generate(self, prompt: str, **kwargs) -> str:
        ...

    def get_token_count(self, text: str) -> int:
        import tiktoken
        enc = tiktoken.get_encoding("cl100k_base")
        return len(enc.encode(text))
```

```python
# app/ai/anthropic.py
from anthropic import AsyncAnthropic
from .base import LLMProvider

class AnthropicClaudeProvider(LLMProvider):
    def __init__(self, api_key: str):
        self.client = AsyncAnthropic(api_key=api_key)

    async def generate(self, prompt: str, **kwargs) -> str:
        response = await self.client.messages.create(
            model=kwargs.get("model", "claude-sonnet-4-20250514"),
            max_tokens=kwargs.get("max_tokens", 1024),
            messages=[{"role": "user", "content": prompt}]
        )
        return response.content[0].text
```

```python
# app/config.py
from pydantic_settings import BaseSettings

    default_model: str = "claude-sonnet-4-20250514"
    max_tokens_per_call: int = 2000
```

### Step 5 — Preprocessing pipeline (one-pass)

One prompt that does cleanup + chunking + extraction in a single AI call:

```
You are a content analyst. Process this transcript in one pass.

TASK:
1. Fix sentence boundaries and remove filler words (um, uh, like, you know)
2. Split into semantic chunks grouped by topic/theme
3. For each chunk, identify: type (hook/claim/story/data/question/cta), content, confidence

OUTPUT FORMAT (JSON array):
[
  {"chunk_type": "hook", "content": "...", "confidence": 0.9},
  {"chunk_type": "claim", "content": "...", "confidence": 0.85},
  ...
]

TRANSCRIPT:
{transcript_text}
```

Parse the JSON response. If parsing fails, fall back to returning the whole transcript as one chunk of type "mixed".

### Step 6 — Generation prompts per platform

Each platform gets its own prompt assembled from:
- The extracted chunks
- The selected template structure
- Platform-specific formatting rules

**LinkedIn:**
```
You are a LinkedIn ghostwriter. Write a post using the {TEMPLATE_NAME} format.

Rules:
- Hook in line 1 (stop the scroll)
- Body follows the template structure
- End with a question or CTA
- No AI-sounding language — write like a human
- 150-300 words

Template sections: {template_sections}

Content atoms extracted from the transcript:
{chunks_formatted}
```

**X thread:**
```
You are a Twitter/X thread writer. Create a 5-post thread.

Rules:
- Each post under 280 characters
- Post 1 is the hook (most important)
- Posts should stand alone if someone doesn't follow the thread
- End post 4 with a cliffhanger, post 5 is the payoff/CTA
- Write like a human, not AI

Content: {chunks}
```

**TikTok:**
```
You are a TikTok scriptwriter. Create 3 hook variants (3-5 seconds each).

Rules:
- Each hook must start with a pattern interrupt (visual or auditory)
- Under 15 words per hook
- Create curiosity or emotion in the first second
- No setup — dive straight in

Content: {transcript_summary}
```

**Reddit:**
```
You are a Reddit post writer. Write a post for the {subreddit} community.

Rules:
- Conversational tone, not corporate
- No emoji
- Include a hook in the title
- Body should invite discussion (end with a question)
- 200-500 words

Content: {chunks}
```

### Step 7 — Quality scoring (heuristic)

No model call — simple rules:

```python
def score_draft(content: str, template_id: str, platform: str) -> float:
    score = 0.5
    # Length check (platform-specific)
    if platform == "linkedin" and 150 <= len(content.split()) <= 300:
        score += 0.15
    elif platform == "x" and len(content) <= 280 * 5:  # thread
        score += 0.15
    elif platform == "tiktok" and len(content.split()) < 50:
        score += 0.15
    # CTA presence
    if any(q in content.lower() for q in ["?", "what do you", "share", "tell me"]):
        score += 0.15
    # Template structure check (basic)
    if template_id == "story" and "once upon" in content.lower():
        score += 0.1
    elif template_id == "pas" and any(w in content.lower() for w in ["problem", "struggle", "challenge"]):
        score += 0.1
    return min(score, 1.0)
```

Score displayed as a badge. No threshold blocking — user sees all drafts.

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Async/queue | None | Synchronous MVP — validate pipeline first |
| Database | SQLite | Local-first, no install overhead |
| Embeddings | None in MVP | Semantic search is v2 |
| Prompt versioning | In-code (env/json) | Schema deferred until workflow proven |
| Quality scoring | Heuristic | No extra AI call needed |
| Reddit posting | Deferred to v2 | Generate drafts only, OAuth later |

---

## Failure Modes

| Failure | Handling | User sees |
|---------|----------|-----------|
| Claude API timeout | Return 500 with error message | "Generation failed — check API key and try again" |
| Malformed AI response | Fall back to raw transcript chunks | Partial output with warning |
| Empty transcript | Validate before sending | "Please paste a transcript first" |
| Transcript too long (>10k words) | Return 400 | "Transcript too long — max 10,000 words" |
